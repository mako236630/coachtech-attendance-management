
# 勤怠管理アプリ 

## 概要
- できること：一般ユーザーが打刻（出勤・休憩・退勤）し、管理者が勤怠を確認できる勤怠管理アプリです
- 工夫した点：休憩を1日複数回取れるように `rests` を分離し、申請ステータスで承認フローを安全に管理しました

## 主な機能
### 一般ユーザー
- 会員登録・ログイン・ログアウト
- 打刻（出勤・休憩開始・終了・退勤）
- 勤怠の修正申請（承認待ち→承認後に反映）

### 管理者
- 勤怠の確認
- 修正申請の承認

このアプリは、スクールでもらった仕様書をもとに未経験の状態から1ヶ月という短期間で仕様を仕上げることを目標に開発しました。
開発にあたっては、単にAIにコードを生成させるのではなく、どうしてこのテーブルが必要なのか、このデータはどうやって画面まで運ぶのかといった根本的な仕組みをAIに質問しながら、自分なりに納得して進めることを大切にしました。

一番苦労したのは、PHPでの勤務時間の計算です。打刻した時間から休憩時間を引いて合計を出すという処理や、時間の文字を計算できる形に変えて、変数に入れて、それを表示させる…という流れを整理するのに何度もつまづきましたが、AIにヒントをもらいながら、なんとか自力で計算のロジックを組み上げることができました。今もまだ完璧に理解できた！とは言えない部分もありますが、今後の課題として学習をしていきたいと思います。
一方で、仕様書通りに動くかどうかを一つずつチェックしていく機能テスト作業が非常にやりがいを感じることができ、テストを通じてバグを未然に防ぎ、システムの品質を保つ重要性と楽しさを実感しています。
これからの課題は、苦手な計算ロジックをもっとスラスラ書けるようになることです。今はPHPで精一杯ですが、今後はJavaScriptを使って画面をパッと切り替えるような工夫もして、使う人が「便利だな」と思えるアプリを作れるようになりたいです。

## 環境構築 

**Dockerビルド** 
```
git clone git@github.com:mako236630/coachtech-attendance-management.git
cd coachtech-attendance-management 
docker-compose up -d --build
```


**Laravel 環境構築** 
```
docker-compose exec php bash
composer install
cp .env.example .env
```
.env は、DB接続部分を修正してください。 
また、Mailtrapの設定をしてください。
```
php artisan key:generate 
php artisan migrate 
php artisan db:seed
```

## ER図  
 - rests(休憩データ): 一日の中で複数回の休憩を可能にする為、休憩データを独立させattendances(勤怠データ)と1対多で紐付けています。
 - 打刻時刻と修正時刻について: 打刻時刻と修正申請中の時刻を別のカラムで管理し、管理者による承認後のデータ更新を安全に行えるようにしています。
 - 申請ステータスの管理: attendances(勤怠データ)にstatusカラムを設け、「0:未申請」「1:承認待ち」「2:承認済み」の状態を管理し管理者が確認・承認したデータのみを正として扱えるようにしました。
